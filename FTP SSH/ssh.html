📉 1. Failed SSH – Attack Reconnaissance

🎯 Top Attacker IPs (Basic)

grep "Failed password" /var/log/auth.log | awk '{print $(NF-3)}' | sort | uniq -c | sort -nr

🎯 Top Attacker IPs (Advanced – precise "from" extraction)

grep -E "sshd.*Failed password" /var/log/auth.log | awk '{for(i=1;i<=NF;i++) if($i=="from") print $(i+1)}' | sort | uniq -c | sort -nr | head -20

👥 Most Targeted Usernames (Brute-force focus)

grep "Failed password" /var/log/auth.log | sed -E 's/.*Failed password for (invalid user )?([^ ]+) from.*/\2/' | sort | uniq -c | sort -nr | head -20

⏰ Attack Peak Timestamps (When are we under siege?)

grep "Failed password" /var/log/auth.log | awk '{print $1 " " $2 " " $3}' | sort | uniq -c | sort -nr | head -20

🤖 10. Complete SSH Automated Analyzer (Production Ready)

#!/bin/bash
LOGFILE=${1:-/var/log/auth.log}
echo "╔════════════════════════════════════════╗"
echo "║   SSH BRUTE FORCE & ACCESS REPORT      ║"
echo "╚════════════════════════════════════════╝"
echo -e "\n[🔥 TOP 15 ATTACKER IPs]"
grep "Failed password" $LOGFILE | awk '{print $(NF-3)}' | sort | uniq -c | sort -nr | head -15

echo -e "\n[👥 TOP 15 TARGETED USERNAMES]"
grep "Failed password" $LOGFILE | sed -E 's/.*Failed password for (invalid user )?([^ ]+) from.*/\2/' | sort | uniq -c | sort -nr | head -15

echo -e "\n[✅ TOP 10 SUCCESSFUL LOGINS by IP]"
grep "Accepted " $LOGFILE | awk '{for(i=1;i<=NF;i++) if($i=="from") print $(i+1)}' | sort | uniq -c | sort -nr | head -10

echo -e "\n[⏰ PEAK ATTACK HOURS]"
grep "Failed password" $LOGFILE | awk '{print $3}' | cut -d: -f1 | sort | uniq -c | sort -nr | head -8

echo -e "\n[⚠️  ANOMALY: IPs that brute-forced AND succeeded]"
comm -12 <(grep "Failed password" $LOGFILE | awk '{print $(NF-3)}' | sort -u) <(grep "Accepted " $LOGFILE | awk '{for(i=1;i<=NF;i++) if($i=="from") print $(i+1)}' | sort -u) | head -10

echo -e "\n[📦 TOTAL FAILED ATTEMPTS] $(grep -c "Failed password" $LOGFILE)"
echo "[🔓 TOTAL SUCCESSFUL LOGINS] $(grep -c "Accepted " $LOGFILE)"

📉 Failed logins per user (including invalid users)

grep "Failed password" /var/log/auth.log | awk '{for(i=1;i<=NF;i++) if($i=="for") {if($(i+1)=="invalid") print $(i+2); else print $(i+1)}}' | sort | uniq -c | sort -nr

🕵️‍♂️ Extract all successful login timestamps per user (insider threat)

grep "Accepted " /var/log/auth.log | awk '{for(i=1;i<=NF;i++) if($i=="for") user=$(i+1); print $1,$2,$3" -> "user}' | sort -k4
⚠️ Critical Security Alert: SSH brute-force attacks often precede lateral movement. Immediately investigate any IP that shows both failed and successful logins. Use `fail2ban` with custom recidive jail.
💡 Pro Tip – Forensic Timeline: Combine `zgrep` for rotated logs: `zgrep "Failed password" /var/log/auth.log*` to analyze weeks of attack patterns. Always correlate SSH logs with `lastlog`, `w`, and `journalctl -u ssh`.
🧠 HTB / CTF Edge: For HackTheBox labs, focus on "invalid user" enumeration — attackers often probe default usernames (root, admin, ubuntu, pi). The ultra advanced one-liner reveals exact recon tactics.
