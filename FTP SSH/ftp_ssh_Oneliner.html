🖥️ 1. SSH Hardening & Intrusion Analysis

🎯 SSH Top IP + User + Time

grep "Failed password" /var/log/auth.log | awk '{print $11" | "$9" | "$1" "$2" "$3}' | sort | uniq -c | sort -nr | head

📋 SSH Specific IP Timeline

ip="192.168.1.10"; grep "Failed password" /var/log/auth.log | grep "$ip" | awk '{print $1,$2,$3,"| USER:",$9}'

🌍 SSH Top Attacker IP (Count)

grep "Failed password" /var/log/auth.log | awk '{print $11}' | sort | uniq -c | sort -nr | head -20

🔗 SSH IP + Username Combo (Brute Force Pairs)

grep "Failed password" /var/log/auth.log | awk '{print $11" -> "$9}' | sort | uniq -c | sort -nr | head -20

⏱️ Burst Attack Detection (Failures per minute)

grep "Failed password" /var/log/auth.log | awk '{print $1" "$2" "$3}' | sort | uniq -c | sort -nr | head

👤 Unique Username Count per IP

grep "Failed password" /var/log/auth.log | awk '{print $11" "$9}' | sort -u | awk '{print $1}' | sort | uniq -c | sort -nr | head

🎭 Detect Password Spraying (Most targeted usernames)

grep "Failed password" /var/log/auth.log | awk '{print $9}' | sort | uniq -c | sort -nr | head

📡 Live Attack Monitor (Real-time tail)

tail -f /var/log/auth.log | grep --line-buffered "Failed password" | awk '{print $1" "$2" "$3" | "$11" | "$9}'

📅 Timeline + IP + User (Full visibility)

grep "Failed password" /var/log/auth.log | awk '{print $1,$2,$3,"| IP:",$11,"| USER:",$9}' | sort | head -50

🌐 Distributed Attack Detection (Same username across many IPs)

grep "Failed password" /var/log/auth.log | awk '{print $9" "$11}' | sort -u | awk '{print $1}' | sort | uniq -c | sort -nr | head

📊 Suspicious IP Score (Attempts + Unique Users)

grep "Failed password" /var/log/auth.log | awk '{print $11" "$9}' | sort | uniq | awk '{ip[$1]++; users[$1]=users[$1]" "$2} END {for (i in ip) {split(users[i],u); print i, ip[i], length(u)}}' | sort -k2 -nr | head

📟 JSON Output for SIEM Integration

grep "Failed password" /var/log/auth.log | awk '{print "{\"ip\":\""$11"\",\"user\":\""$9"\",\"time\":\""$1" "$2" "$3"\"}"}'

📂 2. FTP Service Forensics & Attack Patterns

🔥 FTP Top IP + User + Time

grep "FAIL LOGIN" /var/log/vsftpd.log | awk '{print $(NF)" | "$(NF-3)" | "$1" "$2" "$3}' | sort | uniq -c | sort -nr | head

🏆 FTP Advanced Attacker Ranking (IP -> Username)

grep "FAIL LOGIN" /var/log/vsftpd.log | awk '{print $(NF)" -> "$(NF-3)}' | sort | uniq -c | sort -nr | head

🔓 Anonymous FTP Access Detection

grep -i "anon" /var/log/vsftpd.log | grep "OK LOGIN" | awk '{print $1,$2,$7}' | sort | uniq -c

📤 Data Exfiltration: Top Downloaded Files

grep "i" /var/log/xferlog | awk '{print $9" "$2}' | sort | uniq -c | sort -nr | head -20

📥 Total Download & Upload Size (MB)

echo "Total Download (MB):"; awk '$9=="i" {sum+=10*$7/1024} END {print sum}' /var/log/xferlog
echo "Total Upload (MB):"; awk '$9=="o" {sum+=10*$7/1024} END {print sum}' /var/log/xferlog

🚨 Directory Traversal & Chroot Escape Attempts

grep -i "\/..\/" /var/log/vsftpd.log | awk '{print $7" attempted: "$NF}'
grep -i "chroot" /var/log/vsftpd.log | grep -v "OK"

⚠️ CRITICAL SECURITY ALERT: FTP transmits credentials in plaintext. Always investigate successful logins from unexpected IPs immediately. SSH brute force with common credentials (root/admin) requires immediate blocking.
💡 Pro Tips for Penetration Testers:
• Combine SSH + FTP logs to identify multi-service attackers (lateral movement)
• Use GeoIP to detect anomalous login attempts from non-business countries
• Monitor for password spraying (single username across many IPs)
• Xferlog is your best friend for data exfiltration detection (large downloads = red flag)
• Always correlate failed logins with successful sessions to detect account compromise
🔧 Log File Locations Reference:
SSH: /var/log/auth.log (Debian/Ubuntu) or /var/log/secure (RHEL/CentOS)
FTP: /var/log/vsftpd.log, /var/log/xferlog, /var/log/proftpd/
Journal: journalctl -u ssh -u vsftpd -f
